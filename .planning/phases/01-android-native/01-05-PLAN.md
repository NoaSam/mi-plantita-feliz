Android Studio is required for:
- `npx cap add android` (needs Android SDK)
- Building the APK/AAB
- Running on emulator or connected device

Per user decision, Android Studio is not currently installed.
  </context>
  <how-to-do>
1. Install Android Studio:
```bash
brew install --cask android-studio
```

2. Open Android Studio and complete setup wizard:
   - SDK Manager -> SDK Platforms -> Install API 36 (Android 16) + API 24 (minimum)
   - SDK Manager -> SDK Tools -> Install Android SDK Build-Tools, Platform-Tools
   - JDK 21 is bundled with Android Studio (no separate install needed)

3. Add to `~/.zshrc`:
```bash
export ANDROID_HOME=$HOME/Library/Android/sdk
export PATH=$PATH:$ANDROID_HOME/platform-tools
export PATH=$PATH:$ANDROID_HOME/tools
```

4. Reload shell: `source ~/.zshrc`

5. Verify:
```bash
echo $ANDROID_HOME
adb version
```

<task type="auto">
  <name>Task 2: Generate Android project and build APK</name>
  <files></files>
  <action>
Run the following commands in sequence. All output goes to the gitignored android/ directory.

1. Build the web app for Capacitor:
```bash
npm run build:android
```

2. Add the Android platform (first time only -- creates android/ directory):
```bash
npx cap add android
```

3. Generate Android icons and splash screen from source images:
```bash
npx @capacitor/assets generate --android \
  --iconBackgroundColor '#2D5A27' \
  --splashBackgroundColor '#FDFCF8'
```
This reads from assets/ and writes to android/app/src/main/res/.

4. Sync web build + plugin config to Android project:
```bash
npx cap sync android
```

5. Post-sync smoke test -- verify critical files exist and config is correct BEFORE starting the long Gradle build:
```bash
# Verify web assets were copied to Android project
test -f android/app/src/main/assets/public/index.html && echo "Web assets synced OK" || (echo "FAIL: web assets missing" && exit 1)

# Verify appId in build.gradle
grep "com.miplantitafeliz.app" android/app/build.gradle && echo "AppId OK" || (echo "FAIL: wrong appId" && exit 1)

# Verify targetSdk is exactly 36 (Play Store requires 35+)
grep 'targetSdkVersion = 36' android/variables.gradle && echo "targetSdk OK" || (echo "FAIL: targetSdkVersion is not 36" && exit 1)

# Verify adaptive icons were generated
ls android/app/src/main/res/mipmap-xxhdpi/ && echo "Icons OK" || (echo "FAIL: icons not generated" && exit 1)

# Verify splash was generated
ls android/app/src/main/res/drawable*/splash* && echo "Splash OK" || (echo "FAIL: splash not generated" && exit 1)
```

6. Add camera permission rationale string to Android resources:
Edit `android/app/src/main/res/values/strings.xml` -- add within the `<resources>` tag:
```xml
<string name="PermissionRationaleCamera">Mi jardin necesita acceso a la camara para fotografiar tus plantas e identificarlas.</string>
```

7. Add keystore-related entries to android/.gitignore (defense in depth, since android/ is already gitignored):
```
keystore.properties
*.keystore
*.jks
```

8. Build a debug APK for testing:
```bash
cd android && ./gradlew assembleDebug
```
The output APK will be at: `android/app/build/outputs/apk/debug/app-debug.apk`

9. If an Android device is connected or emulator is running, install and test:
```bash
adb install android/app/build/outputs/apk/debug/app-debug.apk
```

Note: The `android/` directory is gitignored and will NOT be committed. It is regenerated from capacitor.config.ts + dist/ on each developer's machine via `npx cap add android && npx cap sync`.
  </action>

  </verify>

| Boundary | Description |
|----------|-------------|
| APK signing | Debug APK is unsigned; release AAB requires keystore |
| WebView -> Internet | All network calls go through HTTPS (Supabase, AI APIs) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation |
|-----------|----------|-----------|-------------|------------|
| T-01-10 | Information Disclosure | Debug APK | accept | Debug build is for testing only; release build will require proper keystore signing |
| T-01-11 | Information Disclosure | WebView debugging | mitigate | Capacitor 8 disables webContentsDebugging in release builds by default; only debug APK exposes it |
| T-01-12 | Tampering | Cleartext traffic | mitigate | Capacitor defaults to cleartext: false; all API calls use HTTPS |
</threat_model>
