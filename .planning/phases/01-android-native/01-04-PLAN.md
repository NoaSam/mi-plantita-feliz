  <action>
Create the `assets/` directory at project root and generate source images.

1. Create directory:
```bash
mkdir -p assets
```

2. Determine the primary generation approach. Use Node.js with the `canvas` package (or `sharp` if available) as the primary path. If neither is available, use ImageMagick (`convert`). As a last resort, reuse existing PWA icons.

Check which tools are available:
```bash
which convert 2>/dev/null && echo "ImageMagick available" || echo "No ImageMagick"
ls public/pwa-512.png 2>/dev/null && echo "PWA icon available" || echo "No PWA icon"
```

3. Generate the 4 required source images. Since this is a plant identification app with brand color #2D5A27 (dark green) and background #FDFCF8 (warm cream):

**icon-background.png (1024x1024):** Solid color #2D5A27 background. This is the adaptive icon background layer.

**icon-only.png (1024x1024):** If `public/pwa-512.png` exists and contains a usable plant/leaf icon, resize it to 1024x1024 and use it. Otherwise, create a plant/leaf icon on a green (#2D5A27) background programmatically.

**icon-foreground.png (1024x1024):** The plant/leaf graphic on a transparent background. This is the adaptive icon foreground layer. Must have safe zone padding (the center 66% is guaranteed visible; outer 17% may be cropped by different device launchers).

**splash.png (2732x2732):** Centered plant icon/logo on #FDFCF8 background. The center area should contain the brand mark; outer areas should be solid background color since they'll be cropped on different aspect ratios.

4. After generating, verify dimensions of each file:
```bash
# Using file command (always available) to confirm PNG format
file assets/icon-only.png assets/icon-foreground.png assets/icon-background.png assets/splash.png

# If ImageMagick is available, verify exact dimensions:
identify assets/icon-only.png assets/icon-foreground.png assets/icon-background.png assets/splash.png 2>/dev/null || echo "Install ImageMagick for dimension verification, or verify manually"
```

The critical requirement is that files exist at the correct paths with correct dimensions. Visual quality will be verified by the human checkpoint.
