1. Extend `src/lib/platform.ts` -- keep existing `isIOS()`, add `isAndroid()` and `isNative()`:

```typescript
export function isIOS(): boolean {
  return /iPad|iPhone|iPod/.test(navigator.userAgent);
}

export function isAndroid(): boolean {
  return /Android/.test(navigator.userAgent);
}

/** True when running inside a Capacitor native shell (Android or iOS). */
export function isNative(): boolean {
  return !!(
    window as Window & { Capacitor?: { isNativePlatform?: () => boolean } }
  ).Capacitor?.isNativePlatform?.();
}
```

2. Modify `src/main.tsx` -- add Capacitor detection before the SW registration block. The entire `if ("serviceWorker" in navigator)` block must be wrapped with an additional `!isCapacitor` guard:

```typescript
// Skip SW registration inside Capacitor native shell -- SW intercepts
// Capacitor bridge requests and causes "Plugin not implemented" errors.
const isCapacitor = !!(
  window as Window & { Capacitor?: { isNativePlatform?: () => boolean } }
).Capacitor?.isNativePlatform?.();

if ("serviceWorker" in navigator && !isCapacitor) {
  // ... existing SW registration code unchanged ...
}
```

Note: Use the inline `window.Capacitor` check here instead of importing from platform.ts because main.tsx runs before module resolution and we need the check to be synchronous and side-effect free at the top level.

3. Modify `vite.config.ts` -- add conditional `base` path and conditional VitePWA plugin:

```typescript
export default defineConfig(({ mode }) => {
  const isCapacitor = process.env.VITE_CAPACITOR === "true";

  return {
    base: isCapacitor ? "./" : "/",
    server: { host: "::", port: 8080, hmr: { overlay: false } },
    plugins: [
      react(),
      !isCapacitor && VitePWA({
        // ... entire existing VitePWA config unchanged ...
      }),
    ].filter(Boolean),
    resolve: { alias: { "@": path.resolve(__dirname, "./src") } },
  };
});
```

The key changes: (a) `base` is `"./"` when VITE_CAPACITOR=true, `"/"` otherwise; (b) VitePWA plugin is excluded from Android builds via the `!isCapacitor &&` guard. This ensures no service worker is generated in the Android build output.

4. Update `index.html` line 5 -- add `viewport-fit=cover` to the viewport meta tag:
```html
<meta name="viewport" content="width=device-width, initial-scale=1.0, maximum-scale=1.0, user-scalable=no, viewport-fit=cover" />
```

This is required for Android notch/cutout safe area CSS insets to work.
